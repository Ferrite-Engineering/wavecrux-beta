# Captured fixtures — SPI

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

### Block template

```markdown
## `<fixture_name>.fst` — <one-line scenario summary>

- **Source project:** <project name> — <https://github.com/...>
- **Source commit:** `<sha>` (acquired YYYY-MM-DD)
- **License:** <name> (SPDX: `<identifier>`)
- **Author / maintainer:** <name>
- **Original location:** `<path/to/testbench>`
- **Capture method:** <"direct .fst from repo" | "built locally with `iverilog …`" | "built locally with `verilator --trace-fst …`" | "cocotb run: `pytest tb/… `">
- **Acquired:** YYYY-MM-DD
- **Size:** <bytes raw / bytes committed>
- **What it exercises:** <2–3 sentence scenario description>
- **Hand-verified anchors:**
  - <e.g. "write to 0x40, data 0xdeadbeef, at t≈230 ns">
  - <e.g. "burst of 4 reads starting at 0x1000">
- **Snapshot regenerated by:** `dart run tool/regenerate_captured_snapshots.dart --decoder spi --fixture <fixture_name>`
```

## Current captures

## `nandland_spi_master_mode3_loopback.fst` — two back-to-back bytes from a real SPI master, MISO looped back to MOSI

- **Source project:** nandland / spi-master — <https://github.com/nandland/spi-master>
- **Source commit:** `1ab581969f8ea2f794072a144b55cd74e5a54f34` (acquired 2026-05-31)
- **License:** MIT (SPDX: `MIT`)
- **Author / maintainer:** Russell Merrick
- **Original location:** `Verilog/source/SPI_Master.v`,
  `Verilog/source/SPI_Master_With_Single_CS.v`, and
  `Verilog/sim/SPI_Master_With_Single_CS_TB.sv`.
- **Capture method:** built locally with
  `iverilog -g2012 -I. -o tb_spi.vvp SPI_Master_With_Single_CS_TB.sv SPI_Master_With_Single_CS.v && vvp tb_spi.vvp && vcd2fst -v dump.vcd -f nandland_spi_master_mode3_loopback.fst -Z -c`.
  Helpers + reproducible build under `helpers/nandland/` (RTL + testbench vendored
  verbatim, MIT header preserved; `Makefile` is the regenerator entry point).
- **Acquired:** 2026-05-31
- **Size:** 9322 bytes raw VCD / 1153 bytes committed FST.
- **What it exercises:** Russell Merrick's `SPI_Master_With_Single_CS` driven
  by its bundled SystemVerilog testbench (`SPI_MODE=3` → CPOL=1, CPHA=1, sample
  on rising edge for mode-3; `CLKS_PER_HALF_BIT=4`; `MAX_BYTES_PER_CS=2`). The
  testbench wires MISO ← MOSI in loopback and sends two bytes (0xC1 then 0xC2)
  back-to-back inside a single CS-low frame. Covers CS framing, mode-3 sampling,
  MSB-first bit order, and multi-word grouping.
- **Hand-verified anchors:**
  - Single decoded transaction `SPI 2 words`, MOSI = `0xC1 0xC2`,
    MISO = `0xC1 0xC2`, exactly matching the testbench's
    `Sent out 0xC1, Received 0xc1` / `Sent out 0xC2, Received 0xc2` `$display`
    output.
  - CS-low frame spans `t=86` → `t=618` (raw sim units; bundled testbench has
    no `` `timescale ``, so iverilog defaults to 1 s/1 s — the decoder is
    bus-clockless so the absolute number is irrelevant).
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/spi_captured_fixtures_test.dart`
