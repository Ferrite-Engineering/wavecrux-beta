# Captured fixtures — AHB-Lite

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

## Current captures

## `ahb_lite_shalan_dmac.fst` — 5 read+write SINGLE-beat pairs driven by the MS_DMAC_AHBL DMA controller

- **Source project:** Mohamed Shalan / MS_DMAC_AHBL — <https://github.com/shalan/MS_DMAC_AHBL>
- **Source commit:** `d2ea9e38e7ad9392f6675861f8cfcd3ee137c62c` (acquired 2026-05-31)
- **License:** Apache-2.0 (SPDX: `Apache-2.0`)
- **Author / maintainer:** Mohamed Shalan (AUC)
- **Original location:** `hdl/rtl/MS_DMAC_AHBL.v` and `hdl/rtl/ahbl_util.vh`. The upstream `hdl/dv/MS_DMAC_AHBL_tb.v` testbench targets an older RTL revision (uses a `PIRQ` input the current module no longer exposes), so we vendor only the synthesizable RTL + supporting include file and supply our own pure-Verilog testbench.
- **Capture method:** built locally with `iverilog -g2012 -I. -o tb_ahb.vvp tb_ms_dmac_ahbl.v MS_DMAC_AHBL.v && vvp tb_ahb.vvp -fst`. Helpers + reproducible build under `helpers/shalan/`.
- **Acquired:** 2026-05-31
- **Size:** 2053 bytes raw / 2053 bytes committed.
- **What it exercises:** A pure-Verilog testbench drives the DMAC's slave AHB-Lite port through four register writes (SADDR=0x40000000, DADDR=0x50000000, SIZE=4, CTRL=0x12120001 — word-typed transfers with word-stride auto-increment on both ends) followed by a software trigger. The DMAC then marches through 5 word reads from the source region and 5 word writes to the destination region on its master AHB-Lite port. HBURST and HRESP (both required decoder bindings) are tied to constants in the testbench — `SINGLE` / `OKAY` — since the DMAC implements neither bursts nor error responses; the synthetic `generated/` fixtures cover those code paths.
- **Hand-verified anchors:**
  - First read: `R 0x40000000 → 0xC0FFEE00` at t=450 ns. Source word 0 from the testbench memory model.
  - Mid-burst write: `W 0x50000008 = 0xC0FFEE08` at t=650 ns. Dest word 2; data preserved end-to-end through the DMAC.
  - Last write: `W 0x50000010 = 0xC0FFEE10` at t=810 ns. Confirms the DMAC ran one extra transfer beyond SIZE=4 (CNTR runs 0…4 inclusive → 5 transfers, matching the RTL).
- **Note on transfer count:** the upstream CNTR comparison uses `CNTR == SIZE_REG` which gives `SIZE+1` transfers rather than `SIZE`. Pre-existing RTL behavior; the decoder reflects what's on the bus.
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/ahb_lite_captured_fixtures_test.dart`
