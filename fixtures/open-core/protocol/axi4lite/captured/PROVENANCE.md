# Captured fixtures — AXI4-Lite

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

## `forencich_axil_ram.fst` — 9-scenario AXI4-Lite RAM exerciser (writes, reads, stress)

- **Source project:** alexforencich/verilog-axi — <https://github.com/alexforencich/verilog-axi>
- **Source commit:** `516bd5dadc3365b7f9e225d2af8fe0b8d804fe53` (acquired 2026-05-31)
- **License:** MIT (SPDX: `MIT`) — per-file header in `tb/axil_ram/Makefile` and `tb/axil_ram/test_axil_ram.py`
- **Author / maintainer:** Alex Forencich
- **Original location:** `tb/axil_ram/test_axil_ram.py` + `rtl/axil_ram.v`
- **Capture method:** cocotb run from a clean clone of upstream — `cd tb/axil_ram && PATH=~/.venvs/wavecrux-cocotb/bin:$PATH make WAVES=1 SIM=icarus` (first run materializes `iverilog_dump.v`), `patch -p1 < helpers/forencich_axil_ram_aresetn_dump.patch`, then re-run `make WAVES=1 SIM=icarus`. See [`helpers/cocotb_setup.md`](helpers/cocotb_setup.md) for the venv + simulator setup.
- **Acquired:** 2026-05-31
- **Size:** 22327 bytes raw FST (well under 5 MB target; snapshot JSON is ~1 MB / 3712 transactions). The lite protocol's single-beat-per-transaction shape means transaction count scales linearly with byte count, so the snapshot is larger than the FST despite the small trace.
- **What it exercises:** 9 cocotb test scenarios concatenated into one FST — `run_test_write_001..004` (the cross-product of `cycle_pause` idle/backpressure inserters × byte-aligned vs unaligned writes), `run_test_read_001..004` (same cross-product on the read side), and `run_stress_test_001` (256 outstanding random-address, random-length writes interleaved with random-address random-length reads, with bursts of `idle_inserter=cycle_pause` and `backpressure_inserter=cycle_pause`). Every transaction is a single-beat AXI4-Lite read or write — there are no bursts (`awlen`/`arlen` don't exist in the lite protocol), and there's no exclusive access. Coverage value: AXI4-Lite is the open-core baseline AMBA decoder; this fixture covers the same idle-inserter / backpressure-inserter / aligned-vs-unaligned cross-product that the Pro `axi_register` fixture covers for the full protocol, validating that the lite decoder handles the same handshake-stall patterns.
- **Hand-verified anchors:**
  - First transaction at t=70 ns decodes as `W 0x0FFC = 0xAAAAAAAA [OKAY]`, wstrb=0xF — matches the testbench's first write phase (initial value scan with 0xAA bytes at the address boundary between the lower and upper halves of the 16-bit address space).
  - Total transaction count is 3712 across 71.05 µs of sim time, all with `response: "OKAY"` and `isError: false` — the lite protocol doesn't generate SLVERR/DECERR in this DUT (it's a simple RAM with no bus-error path) and Forencich's cocotbext-axi master generates spec-compliant traffic only.
  - Last transaction at t=71050 ns is `R 0x3EF0 = 0x1A191918 [OKAY]` — final read of the 27-byte read-back at the end of the stress test's last iteration, matching cocotb's log line `71060.01ns Read complete addr: 0x00003ed8 ... data: 00 01 02 ... 18 19`. Byte 24 = 0x18, byte 25 = 0x19 (visible in the little-endian decoded word).
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/axi4lite_captured_fixtures_test.dart`
