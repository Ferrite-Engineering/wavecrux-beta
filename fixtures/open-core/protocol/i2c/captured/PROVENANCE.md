# Captured fixtures — I²C

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

### Block template

```markdown
## `<fixture_name>.fst` — <one-line scenario summary>

- **Source project:** <project name> — <https://github.com/...>
- **Source commit:** `<sha>` (acquired YYYY-MM-DD)
- **License:** <name> (SPDX: `<identifier>`)
- **Author / maintainer:** <name>
- **Original location:** `<path/to/testbench>`
- **Capture method:** <"direct .fst from repo" | "built locally with `iverilog …`" | "built locally with `verilator --trace-fst …`" | "cocotb run: `pytest tb/… `">
- **Acquired:** YYYY-MM-DD
- **Size:** <bytes raw / bytes committed>
- **What it exercises:** <2–3 sentence scenario description>
- **Hand-verified anchors:**
  - <e.g. "write to 0x40, data 0xdeadbeef, at t≈230 ns">
  - <e.g. "burst of 4 reads starting at 0x1000">
- **Snapshot regenerated by:** `dart run tool/regenerate_captured_snapshots.dart --decoder i2c --fixture <fixture_name>`
```

## Current captures

## `i2c_forencich_master_slave.fst` — two back-to-back writes from a real i2c_master to an addressed i2c_slave

- **Source project:** Alex Forencich / verilog-i2c — <https://github.com/alexforencich/verilog-i2c>
- **Source commit:** `a65be4045e898a52e791c6ee71f8f79a7cd2e129` (acquired 2026-05-31)
- **License:** MIT (SPDX: `MIT`)
- **Author / maintainer:** Alex Forencich
- **Original location:** `rtl/i2c_master.v` and `rtl/i2c_slave.v`. The repository ships MyHDL/Python stimulus testbenches under `tb/` — we vendor only the synthesizable RTL and supply our own pure-Verilog testbench.
- **Capture method:** built locally with `iverilog -g2012 -o tb_i2c.vvp tb_i2c_master_slave.v i2c_master.v i2c_slave.v && vvp tb_i2c.vvp -fst`. Helpers + reproducible build under `helpers/forencich/`.
- **Acquired:** 2026-05-31
- **Size:** 6213 bytes raw / 6213 bytes committed.
- **What it exercises:** A pure-Verilog testbench wires Forencich's `i2c_master` + `i2c_slave` onto a shared open-drain SDA/SCL bus with explicit `pullup()` resolution. The master is driven with an AXIS command program that performs two back-to-back writes to slave address 0x50: (1) multi-byte 0xAB 0xCD, (2) single-byte 0x42. Covers START / addr+W / per-byte ACK / multi-byte data / STOP / repeated transaction. The slave is configured with `device_address = 0x50` and `device_address_mask = 0x7F` (exact match).
- **Hand-verified anchors:**
  - First transaction: `I²C 0x50 W 2 bytes`, data `0xAB 0xCD`, ACK pattern `ACK ACK`, spans t=180 ns to t≈30 µs.
  - Second transaction: `I²C 0x50 W 0x42`, single-byte ACK, spans t≈30.4 µs to t≈50.6 µs.
  - Bus settles to all-1 between transactions (SDA + SCL idle high on the pull-up).
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/i2c_captured_fixtures_test.dart`
