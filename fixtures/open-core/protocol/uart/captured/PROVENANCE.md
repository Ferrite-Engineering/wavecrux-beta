# Captured fixtures — UART

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

### Block template

```markdown
## `<fixture_name>.fst` — <one-line scenario summary>

- **Source project:** <project name> — <https://github.com/...>
- **Source commit:** `<sha>` (acquired YYYY-MM-DD)
- **License:** <name> (SPDX: `<identifier>`)
- **Author / maintainer:** <name>
- **Original location:** `<path/to/testbench>`
- **Capture method:** <"direct .fst from repo" | "built locally with `iverilog …`" | "built locally with `verilator --trace-fst …`" | "cocotb run: `pytest tb/… `">
- **Acquired:** YYYY-MM-DD
- **Size:** <bytes raw / bytes committed>
- **What it exercises:** <2–3 sentence scenario description>
- **Hand-verified anchors:**
  - <e.g. "write to 0x40, data 0xdeadbeef, at t≈230 ns">
  - <e.g. "burst of 4 reads starting at 0x1000">
- **Snapshot regenerated by:** `dart run tool/regenerate_captured_snapshots.dart --decoder uart --fixture <fixture_name>`
```

## `ben_marshall_tx_9600bps.fst` — Standalone TX testbench, 20 pseudo-random bytes at 9600 bps over a 50 MHz clock

- **Source project:** ben-marshall/uart — <https://github.com/ben-marshall/uart>
- **Source commit:** `5fd2db850a41b65aa34f3a31c663fc8704c7abd8` (acquired 2026-05-30)
- **License:** MIT (SPDX: `MIT`)
- **Author / maintainer:** Ben Marshall
- **Original location:** `rtl/uart_tx.v` + `test/tb_tx.v` (Makefile target `tx`)
- **Capture method:** built locally with `make tx` (iverilog 13.0); produces `work/waves-tx.vcd`, converted to FST via `vcd2fst -v work/waves-tx.vcd -f work/waves-tx.fst -Z -c`. The RTL needs the SV-style `#(parameter ...)` header patch in `helpers/ben_marshall_sv_header.patch` because iverilog 13.0 rejects the original `parameter` declarations that follow the port list while being referenced from it.
- **Acquired:** 2026-05-30
- **Size:** 42 063 347 bytes raw VCD / 13 043 bytes committed FST
- **What it exercises:** Standalone UART transmitter sending 20 bytes back-to-back at 9600 bps with 1 stop bit, no parity, LSB-first. The testbench drives `uart_tx_en` continuously, so bytes group into a single decoded transaction. The fixture validates back-to-back framing at the slowest commonly-deployed UART rate.
- **Hand-verified anchors:**
  - 20-byte burst spans `t=70 ns` (first start bit) to `t≈20.836 ms` (last stop bit), matching 20 × 10 bits × 104.17 µs/bit = 20.834 ms within `vcd2fst` rounding.
  - First byte is `0x24 = '$'`, the same first byte the `tb_rx.v` testbench expects on the receive side — both testbenches share the upstream `$random` seed.
  - All 20 bytes decode without framing or parity errors (the standalone TX cannot inject errors).
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/uart_captured_fixtures_test.dart`

- **Original location:** `rtl/uart_rx.v` + `test/tb_rx.v` (Makefile target `rx`)

- **Acquired:** 2026-05-30
- **Size:** 1 450 206 bytes raw VCD / 2 548 bytes committed FST
- **What it exercises:** Standalone UART receiver decoding 10 bytes driven by the testbench at 115200 bps with 1 stop bit, no parity, LSB-first. Self-checking: tb_rx.v prints `[PASS]` for every byte received as it goes, providing an independent ground truth.
- **Hand-verified anchors:**
  - 10-byte run from `t=9720 ns` to `t=886730 ns`. Each byte = ~87 µs, total ~877 µs (10 × ~87 µs accounting for inter-byte idle gaps within `group_gap_bits = 10`), consistent.

  - All 10 bytes decode without framing errors (the testbench drives clean stop bits at this rate).
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/uart_captured_fixtures_test.dart`

## `ben_marshall_sys_loopback_11520bps.fst` — System loopback testbench, RX→TX echo at 11520 bps over a 50 MHz clock

- **Source project:** ben-marshall/uart — <https://github.com/ben-marshall/uart>
- **Source commit:** `5fd2db850a41b65aa34f3a31c663fc8704c7abd8` (acquired 2026-05-30)
- **License:** MIT (SPDX: `MIT`)
- **Author / maintainer:** Ben Marshall
- **Original location:** `rtl/uart_tx.v` + `rtl/uart_rx.v` + `rtl/impl_top.v` + `test/tb.v` (Makefile target `sys`)
- **Capture method:** built locally with `make sys` (iverilog 13.0); produces `work/waves-sys.vcd`, converted to FST via `vcd2fst -v work/waves-sys.vcd -f work/waves-sys.fst -Z -c`. Requires the same SV-style header patch as the other ben-marshall fixtures.
- **Acquired:** 2026-05-30
- **Size:** 34 296 852 bytes raw VCD / 24 275 bytes committed FST
- **What it exercises:** Full system loopback — the testbench drives 15 bytes into `tb.uart_rxd`, the `impl_top` design wires `uart_rx_data → uart_tx_data` so each received byte is re-transmitted on `tb.i_dut.uart_txd`. Dual-channel decode exercises both `rx` and `tx` bindings in a single trace at an off-standard 11520 bps rate (chosen by the original author to be 1/10 of the typical 115200 baud).
- **Hand-verified anchors:**
  - RX byte sequence decodes as `0x41 0x31 0x42 0x32 0x43 0x33 0x44 0x34 0x00 0x61 0x62 0x63 0x64 0x00 0x00` — i.e. ASCII `"A1B2C3D4\0abcd\0\0"`, exactly matching the 15 `send_byte` invocations printed by `tb.v` (`Sending byte: 65`, `49`, `66`, `50`, …).
  - TX echo decodes as 14 bytes (one short of RX) — the 15th NUL is received at `t≈12.167 ms` but the simulation `$finish` is at `t=13.036 ms`, leaving insufficient time for the loopback path (DUT internal latency + a full 868 µs transmit) to complete the 15th echo. Physically consistent, not a decoder bug.
  - First RX start bit at `t=86 845 ns ≈ 1 × BIT_P` after sim start, matching the testbench's `#BIT_P;` delay before driving `uart_rxd` low.
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/uart_captured_fixtures_test.dart`
