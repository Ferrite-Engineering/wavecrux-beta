# Captured fixtures — SPI flash

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

```markdown

- **Source commit:** `<sha>` (acquired YYYY-MM-DD)
- **License:** <name> (SPDX: `<identifier>`)

- **Capture method:** <"direct .fst from repo" | "built locally with `iverilog …`" | "built locally with `verilator --trace-fst …`" | "cocotb run: `pytest tb/… `">
- **Acquired:** YYYY-MM-DD
- **Size:** <bytes raw / bytes committed>
- **What it exercises:** <2–3 sentence scenario description>
- **Hand-verified anchors:**
  - <e.g. "write to 0x40, data 0xdeadbeef, at t≈230 ns">
  - <e.g. "burst of 4 reads starting at 0x1000">
- **Snapshot regenerated by:** `dart run tool/regenerate_captured_snapshots.dart --decoder spi_flash --fixture <fixture_name>`
```

## Current captures

## `picorv32_spiflash_single_wire.fst` — Reset / Power-Up / READ on Claire Wolf's spiflash behavioral model

- **Source project:** YosysHQ / picorv32 — <https://github.com/YosysHQ/picorv32>
- **Source commit:** `87c89acc18994c8cf9a2311e871818e87d304568` (acquired 2026-05-31)
- **License:** ISC (SPDX: `ISC`)
- **Author / maintainer:** Claire Xenia Wolf
- **Original location:** `picosoc/spiflash.v` (vendored behavioral flash model).
  Upstream's `picosoc/spiflash_tb.v` mixes single-wire SPI with quad / DDR-quad
  reads; our decoder is single-wire only, so we author the testbench locally
  (`tb_picorv32_spiflash_single_wire.v`, SPDX `0BSD`) under `helpers/picorv32/`.
- **Capture method:** built locally with
  `iverilog -o tb.vvp tb_picorv32_spiflash_single_wire.v spiflash.v && vvp tb.vvp && vcd2fst -v tb_picorv32_spiflash_single_wire.vcd -f picorv32_spiflash_single_wire.fst -Z -c`.
  Helpers + reproducible build under `helpers/picorv32/`. `firmware.hex` places
  the byte sequence `93 00 00 00 93 01 00 00` at offset `0x100000`, matching the
  `word0 = 32'h00000093` / `word1 = 32'h00000193` anchors from upstream's
  `spiflash_tb.v`.
- **Acquired:** 2026-05-31
- **Size:** 8781 bytes raw VCD / 1499 bytes committed FST.
- **What it exercises:** Three back-to-back SPI-Flash commands against
  Claire Wolf's behavioral spiflash model — Reset (0xFF) → Power Up
  (0xAB) → READ (0x03) at 24-bit address `0x100000` with 8 data bytes.
  Covers single-byte commands, the JEDEC-spec `incomplete_frame` path
  for bare-0xAB wake-from-deep-power-down (no ID-read tail), and a
  full opcode + 24-bit address + 8-byte data-phase READ. Decoded as a
  stacked decoder on top of the SPI parent (CPOL=0, CPHA=0, MSB-first,
  CS active-low).
- **Hand-verified anchors:**
  - READ data phase decodes to `0x93 0x00 0x00 0x00 0x93 0x01 0x00 0x00`,
    exactly matching upstream `spiflash_tb.v`'s `word0=0x00000093` /
    `word1=0x00000193` little-endian byte order.
  - READ address decodes to `0x100000`, matching upstream's
    `localparam [23:0] offset = 24'h100000`.
  - Reset (0xFF) and Power-Up (0xAB) decoded distinctly (unknown-opcode
    vs. `RES`), matching the `$display("Reset (FFh)")` / `$display("Power Up (ABh)")`
    section markers in upstream's testbench.
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/spi_flash_captured_fixtures_test.dart`
