# Captured fixtures — RISC-V instruction trace

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.


## Current captures

## `riscv_picorv32_wb_ez.fst` — RV32I instruction-fetch trace of the upstream picorv32 `testbench_ez` loop

- **Source project:** YosysHQ / picorv32 — <https://github.com/YosysHQ/picorv32>
- **Source commit:** `87c89acc18994c8cf9a2311e871818e87d304568` (acquired 2026-05-31)
- **License:** ISC (SPDX: `ISC`)
- **Author / maintainer:** Claire Xenia Wolf
- **Original location:** `picorv32.v` (the RV32I core + its `picorv32_wb` Wishbone-master wrapper). The six-instruction loop body is the same one that ships in upstream `testbench_ez.v` (public domain).
- **Capture method:** built locally with `iverilog -g2012 -o tb_picorv32_riscv_ez.vvp tb_picorv32_riscv_ez.v picorv32.v && vvp tb_picorv32_riscv_ez.vvp -fst`. Helpers + reproducible build under `helpers/picorv32/`.
- **Acquired:** 2026-05-31
- **Size:** 5766 bytes raw / 5766 bytes committed (FST already compact at this scale).
- **What it exercises:** A pure-Verilog testbench drives YosysHQ's `picorv32_wb` (Wishbone master) against a 256-word in-memory WB slave preloaded with the upstream `testbench_ez` six-instruction loop body. `picorv32_wb`'s otherwise-tied-off `mem_instr` port is connected and a derived `ifetch_valid = wb_ack & mem_instr` strobe is bound as the RISC-V decoder's `valid` slot — so data load/store transactions on the shared bus are skipped and only completed instruction fetches contribute to the decoded stream. Seven ifetch completions covering one full loop iteration plus the wrap-around fetch.
- **Hand-verified anchors:**
  - PC=0x0000, t=80 ns: `0x3FC00093` → `addi ra, zero, 1020` (the `li ra, 1020` pseudo).
  - PC=0x0014, t=480 ns: `0xFF5FF06F` → `jal zero, …` (-12-byte backward branch).
  - PC=0x0008, t=590 ns: `0x0000A103` → `lw sp, 0(ra)` — the first ifetch of the second iteration, confirming the `jal` landed at `loop:` as expected.
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/riscv_captured_fixtures_test.dart`
- **Note on disassembler output:** the `jal`'s formatted immediate field shows `-2561` rather than the architectural `-12`. This is the documented JKU/Surfer TOML schema behavior — see `verification/VERIFICATION_GUIDE.md` §5.9.10. The PC chain is correct (next ifetch lands at PC=0x08); only the displayed immediate departs from architectural byte units, identically to Surfer.
