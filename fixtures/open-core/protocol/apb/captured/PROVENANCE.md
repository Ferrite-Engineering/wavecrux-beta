# Captured fixtures — APB

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

### Block template

```markdown
## `<fixture_name>.fst` — <one-line scenario summary>

- **Source project:** <project name> — <https://github.com/...>
- **Source commit:** `<sha>` (acquired YYYY-MM-DD)
- **License:** <name> (SPDX: `<identifier>`)
- **Author / maintainer:** <name>
- **Original location:** `<path/to/testbench>`
- **Capture method:** <"direct .fst from repo" | "built locally with `iverilog …`" | "built locally with `verilator --trace-fst …`" | "cocotb run: `pytest tb/… `">
- **Acquired:** YYYY-MM-DD
- **Size:** <bytes raw / bytes committed>
- **What it exercises:** <2–3 sentence scenario description>
- **Hand-verified anchors:**
  - <e.g. "write to 0x40, data 0xdeadbeef, at t≈230 ns">
  - <e.g. "burst of 4 reads starting at 0x1000">
- **Snapshot regenerated by:** `dart run tool/regenerate_captured_snapshots.dart --decoder apb --fixture <fixture_name>`
```

## Current captures

## `apb_axil2apb.fst` — AXI-Lite → APB bridge traffic from ZipCPU/wb2axip's axil2apb + apbslave

- **Source project:** WB2AXIP — <https://github.com/ZipCPU/wb2axip>
- **Source commit:** `df8e7649acf68544a152e39a4589c8e894a4cd0b` (acquired 2026-05-31)
- **License:** Apache License 2.0 (SPDX: `Apache-2.0`)
- **Author / maintainer:** Dan Gisselquist, Ph.D. — Gisselquist Technology, LLC
- **Original location:** `rtl/axil2apb.v`, `rtl/apbslave.v`, `rtl/skidbuffer.v`
- **Capture method:** `cd helpers/apb && make install` (vendored copies of
  the three RTL modules + our `tb_apb.v` testbench, built with
  `iverilog -g2012` and dumped via `vvp tb_apb.vvp -fst`)
- **Acquired:** 2026-05-31
- **Size:** ~2.6 kB committed
- **What it exercises:** Ten APB transactions produced by routing an
  AXI-Lite stimulus stream through Gisselquist's `axil2apb` bridge into
  his `apbslave` peripheral: four full-width writes (`0x100`–`0x10C`),
  four readbacks, one partial-strobe write (`PWSTRB=0x3` updating only
  bytes [1:0] of `0x100`), and one final read confirming the
  byte-strobe merge. `apbslave` is always-ready so every transaction
  completes in two cycles (SETUP + ACCESS); PSLVERR is tied low so all
  responses are OKAY. This corpus complements the synthetic
  `apb_basic` fixture by exercising the decoder against a real
  upstream APB master's idle-cycle distribution and byte-strobe
  semantics.
- **Hand-verified anchors:**
  - `W 0x100 = 0xDEADBEEF` at t=85 ns — first write through the
    bridge, full strobe `0xF`.
  - `R 0x100 → 0xDEADBEEF` at t=325 ns — readback proves the
    bridge → slave → bridge → master path round-trips correctly.
  - `R 0x100 → 0xDEADC0DE` at t=625 ns — readback after partial
    write (`0xC0DE` with `PWSTRB=0x3`) proves PWSTRB byte-merge
    semantics: upper half `0xDEAD` preserved, lower half overwritten
    with `0xC0DE`.
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/apb_captured_fixtures_test.dart`
