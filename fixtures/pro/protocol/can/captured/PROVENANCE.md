# Captured fixtures — CAN / CAN-FD

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.


## `wavecrux_can_exerciser.fst` — Classical CAN 2.0A Standard Data Frame, ID 0x123, data 0xAB 0xCD

- **Source project:** WaveCrux Pro capture pipeline (in-house exerciser)
- **Source commit:** _N/A — original file authored in this commit_
- **License:** public domain (SPDX: `CC0-1.0`)
- **Author / maintainer:** WaveCrux Pro
- **Original location:** `helpers/wavecrux/tb_can_exerciser.v` (in this directory).
- **Capture method:** built locally with
  `iverilog -g2005 -o tb.vvp tb_can_exerciser.v && vvp tb.vvp && vcd2fst -v tb_can_exerciser.vcd -f wavecrux_can_exerciser.fst -Z -c`.
  Helpers + reproducible build under `helpers/wavecrux/`. Same rationale as
  the Avalon / JTAG / PCIe-TLP exercisers — OSS CAN cores live behind
  GPL/LGPL or Bosch refmodel licenses incompatible with the corpus allow-list,
  so we author the trace from the public ISO 11898-1 standard under CC0-1.0.
- **Acquired:** 2026-05-31
- **Size:** _≈2 KB_ raw VCD / 822 bytes committed FST.
- **What it exercises:** A complete Classical CAN 2.0A Standard Data
  Frame on `can_rx`: SOF → 11-bit ID `0x123` → RTR=0 → IDE=0 →
  r0=0 → FDF=0 → DLC=2 → 16 data bits `0xAB 0xCD` → CRC15 (computed
  matching the decoder's polynomial / shift convention) → CRC delimiter
  → ACK (dominant, simulating a listening node) → ACK delimiter →
  7-bit EOF → 3-bit IFS. Bit timing 500 kbit/s (2 µs / bit). Covers
  the bit-stream parser's de-stuffed application-layer signal
  convention, all classical CAN 2.0A fields, the CRC15 validation
  path (no error), and the ACK / EOF / IFS framing tail.
- **Two non-obvious conventions** (matching the bundled generator at
  `tool/generate_can_fixtures.dart`):
  - **De-stuffed application-layer signal.** Per the comment in
    `lib/services/decoders/can_decoder.dart` (the "bit stuffing note"
    section), the decoder reads the controller's logical signal, not
    the raw physical bus. The exerciser drives raw logical bits 1-to-1
    with no stuff-bit insertion.
  - **Explicit FDF bit.** The decoder reads two reserved bits after IDE
    (`r0` then `fdfBit`) to disambiguate classical CAN (`fdfBit=0`)
    from CAN-FD (`fdfBit=1`). Real classical CAN 2.0A wire format only
    has one reserved bit; the generator emits an explicit FDF=0 bit
    (see `generate_can_fixtures.dart` line 80) and the exerciser
    follows suit.
- **Hand-verified anchors:**
  - Frame decodes as `DATA 0x123 [2]` with `data="AB CD"` and
    `isError=false`, exactly matching the testbench's
    `FRAME_ID = 11'h123`, `DLC = 4'd2`, `DATA = 16'hABCD` constants.
  - Frame timeline: SOF at `t=16 000 000` ps (= 16 µs, after the
    8-bit-time IFS gap), end at `t=138 000 000` ps. Span = 122 µs = 61
    bit periods, matching the 61 logical bits between SOF (start) and
    last IFS (inclusive).
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/can_captured_fixtures_test.dart`
