# Captured fixtures — USB 2.0

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.


## Current captures

## `ulixxe_usb_cdc_host_tx.fst` — Full-Speed USB 2.0 host TX: SOF → SETUP → DATA0(GET_DESCRIPTOR) → ACK

- **Source project:** ulixxe / usb_cdc — <https://github.com/ulixxe/usb_cdc>
- **Source commit:** `6798bf42d43be81368d4e5e2c26b262b41e9fb6a` (acquired 2026-05-31)
- **License:** MIT (SPDX: `MIT`) — per-file header in `LICENSE`.
- **Author / maintainer:** ulixxe
- **Original location:** `examples/common/hdl/usb_tasks.v`,
  `examples/common/hdl/usb_rx_tasks.v`, `examples/common/hdl/sim_tasks.v`.
- **Capture method:** built locally with
  `iverilog -g2005 -I . -o tb.vvp tb_usb_cdc_host_tx.v && vvp tb.vvp && vcd2fst -v tb_usb_cdc_host_tx.vcd -f ulixxe_usb_cdc_host_tx.fst -Z -c`.
  Helpers + reproducible build under `helpers/ulixxe/`. ulixxe's host-TX
  tasks (`sof_tx`, `token_tx`, `data_tx`, `handshake_tx`) generate
  NRZI-encoded, bit-stuffed, CRC-computed USB 2.0 wire packets
  byte-for-byte per the spec. Our 0BSD testbench
  (`tb_usb_cdc_host_tx.v`) wires `dp_force` / `dn_force` through
  pull-up/pull-down to `dp_sense` / `dn_sense`, declares the constants
  the tasks expect, and walks a four-packet sequence.
- **Acquired:** 2026-05-31
- **Size:** 7592 bytes raw VCD / 1208 bytes committed FST.
- **What it exercises:** Four canonical USB 2.0 Full-Speed packets
  back-to-back: (1) SOF for frame 0x123, (2) SETUP token to address 0 /
  endpoint 0, (3) DATA0 carrying the 8-byte GET_DESCRIPTOR(DEVICE)
  setup packet (`bmRequestType=0x80, bRequest=0x06, wValue=0x0100,
  wIndex=0, wLength=0x0040`), (4) ACK handshake. Covers SOF / token /
  data / handshake PID decoding, address+endpoint extraction, frame
  number extraction, 8-byte data payload reconstruction, CRC5 + CRC16
  validation, and the bit-stuffed NRZI signal-recovery path on
  real-world wire timing.
- **Hand-verified anchors:**
  - SOF frame field decodes to `291` = `0x123` ✓, matching the
    testbench's `sof_tx(11'h123, ...)` call.
  - SETUP token decodes `addr=0x00 ep=0` ✓, matching
    `token_tx(PID_SETUP, 7'h00, 4'h0, ...)`.
  - DATA0 payload decodes to `0x80 0x06 0x00 0x01 0x00 0x00 0x40 0x00`,
    byte-for-byte matching the GET_DESCRIPTOR(DEVICE) setup packet
    constructed in the testbench.
  - ACK terminator decoded distinctly with `pidName = "ACK"`,
    matching `handshake_tx(PID_ACK, ...)`.
- **Note on `bit_period_ticks`:** The fixture pins
  `bit_period_ticks: 83000` (in ps) rather than auto-deriving (which
  would yield 83 333 ps for 12 Mb/s). iverilog's integer-`time` rounds
  1000.0/12.0 ns → 83 ns, so the actual on-the-wire bit period is
  83 000 ps. The 0.4% gap from the auto-derived 83 333 ps is within
  USB tolerance but tight enough to confuse the decoder's strict SE0
  EOP window; pinning the actual rounded value lets the decoder track
  the trace exactly.
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/usb_captured_fixtures_test.dart`
