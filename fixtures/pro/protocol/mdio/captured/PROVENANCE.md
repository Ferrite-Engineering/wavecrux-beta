# Captured fixtures — MDIO

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

## `mdio_forencich_clause22.fst` — Clause-22 MDIO, 2× write + 2× read against a per-bit slave model

- **Source project:** alexforencich/verilog-ethernet — <https://github.com/alexforencich/verilog-ethernet>
- **Source commit:** `77320a9471d19c7dd383914bc049e02d9f4f1ffb` (acquired 2026-05-30)
- **License:** MIT (SPDX: `MIT`) — per-file header on `example/VCU118/fpga_1g/rtl/mdio_master.v`
- **Author / maintainer:** Alex Forencich
- **Original location:** `example/VCU118/fpga_1g/rtl/mdio_master.v` (vendored verbatim as [`helpers/forencich_mdio_master.v`](helpers/forencich_mdio_master.v))
- **Capture method:** standalone Icarus-Verilog testbench under [`helpers/`](helpers/) — Forencich's `mdio_master.v` is the DUT; [`helpers/tb_mdio_master.v`](helpers/tb_mdio_master.v) drives four Clause-22 transactions and models a per-bit MDIO slave that responds to reads. Rebuild via `cd helpers && make && make install`. No cocotb / Python — pure Verilog initial-block stimulus, so this fixture sidesteps the cocotb-2.0 patch tax that the RGMII / future MII / RMII / GMII captures pay.
- **Acquired:** 2026-05-30
- **Size:** 3298 bytes raw FST (well under 5 MB target; snapshot JSON ≈1.4 KB)
- **What it exercises:** the testbench issues, in order: (T1) write PHY 5, reg 0x00 = 0x1140 (Clause-22 BMCR with 1000Mbps + auto-neg enable bit pattern); (T2) read PHY 5, reg 0x01, slave responds 0x796D (BMSR-style status word); (T3) write PHY 1, reg 0x09 = 0x0200 (1000BASE-T-control register pattern); (T4) read PHY 1, reg 0x02, slave responds 0x0141 (Marvell-OUI-high PHY identifier). All four frames carry the canonical 32-bit preamble + ST=01 + OP/PHYAD/REGAD/TA/DATA layout that the decoder's auto-clause detector should classify as Clause 22. The slave drives `0` on TA[1] and shifts response data MSB-first during the 16 data cycles of each read, exercising the decoder's master-tri-state / slave-drive turnaround handling — the failure mode the RGMII pilot's lesson-#2 watch-out flagged for shared-convention bugs.
- **Hand-verified anchors:**
  - T1 decodes as `W 0x05.0x00 = 0x1140` (`label`), `frame_hex = 0x52821140` = `01 01 00101 00000 10 0001_0001_0100_0000` — exactly matches the stimulus block in `tb_mdio_master.v`. ST and OP are auto-detected as Clause-22 / write.
  - T2 decodes as `R 0x05.0x01 → 0x796D` with no `Bad turnaround — slave did not drive` error — proves the slave's TA[1]=0 drive landed inside the master's tri-state window. The decoded `data` field round-trips the slave's staged `slave_response_data` register byte-for-byte.
  - T4's read data `0x0141` matches the second `slave_response_data` stage value, and PHYAD/REGAD `0x01.0x02` match the second read's command. No errors flagged anywhere in the 21.1 µs trace.
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/mdio_captured_fixtures_test.dart`
