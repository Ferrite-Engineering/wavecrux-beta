# Captured fixtures — Avalon-MM

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

## Current captures

## `wavecrux_avalon_mm_exerciser.fst` — Single-word + partial-byteenable + error-response Avalon-MM traffic

- **Source project:** WaveCrux Pro capture pipeline (in-house exerciser)
- **Source commit:** _N/A — original file authored in this commit_
- **License:** public domain (SPDX: `CC0-1.0`)
- **Author / maintainer:** WaveCrux Pro
- **Original location:** `helpers/wavecrux/tb_avalon_mm_exerciser.v` (in this directory).
- **Capture method:** built locally with
  `iverilog -g2012 -o tb.vvp tb_avalon_mm_exerciser.v && vvp tb.vvp && vcd2fst -v tb_avalon_mm_exerciser.vcd -f wavecrux_avalon_mm_exerciser.fst -Z -c`.
  Helpers + reproducible build under `helpers/wavecrux/` (RTL + Makefile).
  No external sources are vendored — the exerciser is original code
  implementing the protocol defined in the public Intel "Avalon Interface
  Specifications" (document MNL-AVABUSREF). Avalon-MM has no
  permissively-licensed Verilog testbench in the OSS ecosystem (see
  `helpers/wavecrux/README.md` for the rationale).
- **Acquired:** 2026-05-31
- **Size:** 3984 bytes raw VCD / 835 bytes committed FST.
- **What it exercises:** Five back-to-back Avalon-MM transfers against a
  small pipelined slave (1-cycle waitrequest, 2-cycle readdatavalid):
  full-word write `0xDEADBEEF` → `0x000`, round-trip read, partial-byteenable
  write `0x12345678` with `be=4'b0011` → `0x010`, round-trip read (returning
  `0x5678` — upper half remained zero), erroring read to `0x400` returning
  `SLAVE_ERROR`. Covers the master / slave handshake, byteenable masking,
  the OKAY response code, and the SLAVE_ERROR response code in one trace.
- **Hand-verified anchors:**
  - WRITE @ t≈65 ns: `address=0x0`, `writedata=0xdeadbeef`, `be=1111`,
    matching the `do_write(12'h000, 32'hDEADBEEF, 4'b1111)` call in the
    testbench.
  - READ @ t≈105–125 ns: `address=0x0` returns `0xdeadbeef`, confirming
    the slave's 32-bit storage and 2-cycle readdatavalid latency.
  - Partial WRITE @ t≈155 ns: `be=0011` (lower 16 bits only), then READ
    returns `0x5678` (upper half remained at reset value `0x0000`),
    exercising the decoder's `byteenable_bin` field.
  - Erroring READ @ t≈245–265 ns: `address=0x400` returns SLAVE_ERROR
    (`response=2'b10`), demonstrating the decoder's error-response path.
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/avalon_mm_captured_fixtures_test.dart`
