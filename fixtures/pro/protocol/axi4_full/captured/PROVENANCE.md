# Captured fixtures — AXI4 Full

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

The toolchain setup (Python 3.13 venv, cocotb + cocotbext-axi install,
icarus-verilog) is documented in [`helpers/cocotb_setup.md`](helpers/cocotb_setup.md).
The `aresetn` Verilog dump patch shared by every Forencich-sourced fixture
lives at [`helpers/forencich_aresetn_dump.patch`](helpers/forencich_aresetn_dump.patch).

## `forencich_axi_ram_full.fst` — 2-scenario AXI4 RAM exerciser (run_test_write_001 + run_test_read_001)

- **Source project:** alexforencich/verilog-axi — <https://github.com/alexforencich/verilog-axi>
- **Source commit:** `516bd5dadc3365b7f9e225d2af8fe0b8d804fe53` (acquired 2026-05-30, re-captured 2026-05-31)
- **License:** MIT (SPDX: `MIT`) — per-file header in `tb/axi_ram/Makefile` and `tb/axi_ram/test_axi_ram.py`
- **Author / maintainer:** Alex Forencich
- **Original location:** `tb/axi_ram/test_axi_ram.py` + `rtl/axi_ram.v`
- **Capture method:** cocotb run from a clean clone of upstream — `cd tb/axi_ram && patch -p1 < helpers/forencich_aresetn_dump.patch && PATH=~/.venvs/wavecrux-cocotb/bin:$PATH TESTCASE=run_test_write_001,run_test_read_001 make WAVES=1 SIM=icarus` (see [`helpers/cocotb_setup.md`](helpers/cocotb_setup.md)).
- **Acquired:** 2026-05-30 (re-captured 2026-05-31 with `TESTCASE=...` subsetting to meet the 500 KB per-fixture snapshot budget)
- **Size:** 33 445 bytes FST / 302 KB snapshot JSON. The original 25-scenario sweep produced a 688 225 byte FST and a 5.1 MB snapshot — exceeding the 500 KB per-snapshot target by 10×. The other 23 sub-tests are part of the original run's coverage envelope but are exercised in the generated/ corpus instead.
- **What it exercises:** 2 cocotb test scenarios from
  `test_axi_ram.run_test_write_001` and `test_axi_ram.run_test_read_001`,
  selected via `TESTCASE=run_test_write_001,run_test_read_001`. The first
  iterates over every supported `(id, length, addr-offset)` triple for
  `awsize = max_burst_size` with both `idle_inserter=None` and
  `backpressure_inserter=None`. The second does the same for reads. Together
  they produce 464 transaction records — sufficient to exercise the AXI4
  burst-write, burst-read, ID-tagged completion, offset-address, and
  length-sweep paths without the 10× snapshot blow-up of the full 25-test sweep.
- **Hand-verified anchors:**
  - First write: `W 0x00001000 ×3 INCR [OKAY]`, three-beat burst (matches cocotb `run_test_write_001` driving an INCR burst from 0x1000).
  - First read: `R 0x00001003 ×2 INCR [OKAY]`, two-beat burst from offset 0x1003 (matches cocotb `run_test_read_001` reading the unaligned address back).
  - Last write: `W 0x00002000 ×256 INCR [OKAY]` — confirms the 256-beat
    maximum-burst-length path closes cleanly at the end of `run_test_write_001`.
  - Last read: `R 0x00002000 ×256 INCR [OKAY]` — matching 256-beat burst-read
    confirms the read side of the maximum-length sweep.
- **Snapshot regenerated by:** `TESTCASE=run_test_write_001,run_test_read_001 PATH=~/.venvs/wavecrux-cocotb/bin:$PATH make WAVES=1 SIM=icarus` (in upstream `tb/axi_ram/`), then `REGENERATE=1 flutter test test/services/decoders/axi4_full_captured_fixtures_test.dart`.

## `forencich_axi_register.fst` — 25-scenario AXI4 register-stage exerciser

- **Source project:** alexforencich/verilog-axi — <https://github.com/alexforencich/verilog-axi>
- **Source commit:** `516bd5dadc3365b7f9e225d2af8fe0b8d804fe53` (acquired 2026-05-30)
- **License:** MIT (SPDX: `MIT`) — per-file header in `tb/axi_register/Makefile` and `tb/axi_register/test_axi_register.py`
- **Author / maintainer:** Alex Forencich
- **Original location:** `tb/axi_register/test_axi_register.py` + `rtl/axi_register.v` (+ `axi_register_wr.v`, `axi_register_rd.v`)
- **Capture method:** cocotb run from a clean clone of upstream — `cd tb/axi_register && patch -p1 < helpers/forencich_axi_register_aresetn_dump.patch && PATH=~/.venvs/wavecrux-cocotb/bin:$PATH make WAVES=1 SIM=icarus` (see [`helpers/cocotb_setup.md`](helpers/cocotb_setup.md))
- **Acquired:** 2026-05-30
- **Size:** 154632 bytes raw FST (well under 5 MB target; snapshot JSON is 2.9 MB)
- **What it exercises:** 25 cocotb test scenarios from `test_axi_register.run_test_write_001..012`, `run_test_read_001..012`, `run_stress_test_001`. Sweeps the cross-product of `(write / read)` × `(no-idle / idle-cycle inserter)` × `(no-backpressure / backpressure-cycle inserter)` × `(default size, 1-byte, 2-byte)`. The stress test interleaves 256 outstanding transactions through the register-stage pipeline (`REG_TYPE=2`, the 2-deep pipeline configuration) to verify ID-tagged out-of-order completion. The decoder produces 2872 transaction records — fewer than `axi_ram` because `axi_register` is single-beat-per-write rather than burst-oriented (still exercises pipeline behaviour under idle / backpressure stress).
- **Hand-verified anchors:**
  - First write: `W 0x00001004 ×1 INCR [OKAY]` at t=60000 ps, ID=0x0 — single-beat write to offset 0x1004, matching `run_test_write_001` driving an INCR-1 burst with `idle_inserter=None`, `backpressure_inserter=None`.
  - Second write: `W 0x00001005 ×1 INCR [OKAY]` at t=120000 ps, ID=0x1 — offset increments by 1 byte per transaction and ID increments by 1 (default `run_test_write_001` loop pattern).
  - Last stress transaction: `R 0x000078D9 ×79 INCR [OKAY]` at t≈4.083 ms, ID=0xFF — confirms the stress test's ID-tagged out-of-order completion through the register pipeline closes cleanly at the end of `run_stress_test_001`.
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/axi4_full_captured_fixtures_test.dart`

This is the second AXI4 Full captured fixture, providing pipeline-stage register-fabric coverage to complement the `axi_ram` fixture's RAM-array burst coverage. Together they exercise INCR bursts of length 1..256, aligned and unaligned addresses, ID-tagged out-of-order completion under both register-stage and RAM backends, and the cocotbext-axi extensions' idle / backpressure injection patterns.
