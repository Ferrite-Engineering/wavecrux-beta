# Captured fixtures — JTAG

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

## `wavecrux_jtag_tap_exerciser.fst` — IEEE 1149.1 TAP traversal: IR/DR sweep through IDCODE + BYPASS

- **Source project:** WaveCrux Pro capture pipeline (in-house exerciser)
- **Source commit:** _N/A — original file authored in this commit_
- **License:** public domain (SPDX: `CC0-1.0`)
- **Author / maintainer:** WaveCrux Pro
- **Original location:** `helpers/wavecrux/tb_jtag_tap_exerciser.v` (in this directory).
- **Capture method:** built locally with
  `iverilog -g2012 -o tb.vvp tb_jtag_tap_exerciser.v && vvp tb.vvp && vcd2fst -v tb_jtag_tap_exerciser.vcd -f wavecrux_jtag_tap_exerciser.fst -Z -c`.
  Helpers + reproducible build under `helpers/wavecrux/`. Same rationale as
  the Avalon exercisers: most OSS JTAG TAPs live inside larger debug-module
  repos under heavier license footprints (Solderpad, LGPL) or under
  SystemVerilog constructs iverilog can't compile, so the corpus tier is
  satisfied via a CC0-1.0 in-house exerciser implementing the protocol from
  the public IEEE 1149.1 standard.
- **Acquired:** 2026-05-31
- **Size:** 5428 bytes raw VCD / 1081 bytes committed FST.
- **What it exercises:** A complete IEEE 1149.1 TAP traversal — 5 TMS=1
  clocks to reach Test-Logic-Reset, then two full IR-load + DR-shift
  sequences. First load IDCODE (LSB-first `0001` = MSB-first `0x8`) and
  shift 32 bits out of a 32-bit IDCODE register holding `0xDEADBEEF`.
  Then load BYPASS (LSB-first `1111` = MSB-first `0xF`) and shift 4 bits
  through the 1-bit bypass register. Returns to TLR via 5 more TMS=1
  clocks. Covers TAP state-machine traversal, IR/DR scan paths, register
  swap between IDCODE and BYPASS, and bypass-register pass-through.
- **Hand-verified anchors:**
  - First IR transaction: `IR = 0x8` (MSB-first 4-bit value `1000`) at
    t≈110–170 ns, which is `0x1` in LSB-first IEEE 1149.1 encoding — the
    IDCODE instruction.
  - Second IR transaction: `IR = 0xF` (`1111`) at t≈580–640 ns — the
    BYPASS instruction.
  - BYPASS DR shift at t≈670–730 ns: TDI `0x5` (MSB-first `0101` = LSB-first
    `1010`) matches the driver's `bypass_pattern = 4'b1010`. TDO trails TDI by
    one cycle through the bypass register (TDO = `0x4` MSB-first = `0010`
    LSB-first), which is the expected single-cycle delay of the BYPASS
    register.
- **Driver off-by-one note:** The driver samples TDO *after* each
  rising edge → shift, so the captured TDO is one cycle delayed
  relative to the on-the-wire value. This shifts the IDCODE DR result
  by one bit but does not change the decoder's TAP state-tracking,
  which is the primary test material.
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/jtag_captured_fixtures_test.dart`
