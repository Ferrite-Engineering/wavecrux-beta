# Captured fixtures — PCIe TLP

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

```markdown
## `<fixture_name>.fst` — <one-line scenario summary>

- **Source project:** <project name> — <https://github.com/...>
- **Source commit:** `<sha>` (acquired YYYY-MM-DD)
- **License:** <name> (SPDX: `<identifier>`)
- **Author / maintainer:** <name>
- **Original location:** `<path/to/testbench>`
- **Capture method:** <"direct .fst from repo" | "built locally with `iverilog …`" | "built locally with `verilator --trace-fst …`" | "cocotb run: `pytest tb/… `">
- **Acquired:** YYYY-MM-DD
- **Size:** <bytes raw / bytes committed>
- **What it exercises:** <2–3 sentence scenario description>
- **Hand-verified anchors:**
  - <e.g. "write to 0x40, data 0xdeadbeef, at t≈230 ns">
  - <e.g. "burst of 4 reads starting at 0x1000">
- **Snapshot regenerated by:** `dart run tool/regenerate_captured_snapshots.dart --decoder pcie_tlp --fixture <fixture_name>`
```

## Current captures

## `wavecrux_pcie_tlp_exerciser.fst` — MWr32 / MRd32 / CplD / CfgWr0 TLP sweep on 32-bit DW interface

- **Source project:** WaveCrux Pro capture pipeline (in-house exerciser)
- **Source commit:** _N/A — original file authored in this commit_
- **License:** public domain (SPDX: `CC0-1.0`)
- **Author / maintainer:** WaveCrux Pro
- **Original location:** `helpers/wavecrux/tb_pcie_tlp_exerciser.v` (in this directory).
- **Capture method:** built locally with
  `iverilog -g2005 -o tb.vvp tb_pcie_tlp_exerciser.v && vvp tb.vvp && vcd2fst -v tb_pcie_tlp_exerciser.vcd -f wavecrux_pcie_tlp_exerciser.fst -Z -c`.
  Helpers + reproducible build under `helpers/wavecrux/`. PCIe TLP packet
  shapes follow the public PCIe Base Specification (PCI-SIG, §2.2 TLP
  header formats); the corpus tier is satisfied via the CC0-1.0
  in-house exerciser of the documented packet formats. Corundum is the
  obvious upstream candidate (Apache-2.0) but its TLP layer is buried
  inside a multi-thousand-line SoC, not vendorable as a self-contained
  testbench.
- **Acquired:** 2026-05-31
- **Size:** _≈3 KB_ raw VCD / 595 bytes committed FST.
- **What it exercises:** Four canonical TLP packets back-to-back on the
  32-bit DW AXIS-style TLP-streaming interface: MWr32 (Memory Write
  32-bit, address `0x10000000`, payload `0xDEADBEEF`), MRd32 (Memory
  Read 32-bit, address `0x20000000`, tag `0x06`), CplD (Completion with
  Data, `SC` status, payload `0xCAFEBABE`, requester tag matches the MRd
  tag), CfgWr0 (Configuration Write Type 0, target ID `02:00.0`,
  register `0x001`, payload `0x00000506`). Covers fmt/type decoding for
  all four major TLP categories, requester/completer ID extraction, tag
  matching across an MRd→CplD round trip, length-DW + byte-enable
  decoding, address extraction for both Memory and Configuration
  transactions, and the completion-status field decode.
- **Hand-verified anchors:**
  - MWr32 at t≈55 ns: `address=0x10000000`, `data=0xDEADBEEF`,
    `requester_id=01:00.0`, `tag=0x05`, `first_be=1111` — exactly the
    fields encoded in the testbench's first four `send_beat()` calls.
  - MRd → CplD round trip: MRd at t≈115 ns has `tag=0x06`; CplD at
    t≈165 ns has matching `requester_id=01:00.0` and `tag=0x06`, with
    `byte_count=4` and `data=0xCAFEBABE`. Demonstrates tag-based
    completion matching.
  - CfgWr0 at t≈225 ns: `target_id=02:00.0`, `register=0x001`,
    `data=0x00000506` — the `0x506` payload would enable Memory-Space +
    Bus-Master + SERR# in PCIe configuration space register 0x04
    (Command), which lives at DW1 (= register `0x001` in DW-addressed
    terms).
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/pcie_tlp_captured_fixtures_test.dart`
