# Captured fixtures — Avalon-ST

This directory holds traces acquired or produced from public open-source
projects. They complement the deterministic synthetic fixtures next door in
`../generated/` by exercising the decoder against real-world bus traffic.

## License allow-list

Captured fixtures must come from projects under one of these licenses:

- MIT
- BSD-2-Clause / BSD-3-Clause
- Apache-2.0
- ISC
- CC0-1.0
- public domain

GPL/AGPL/proprietary captures are excluded by policy. The static test
`test/static/captured_fixture_licenses_test.dart` walks every
`PROVENANCE.md` in the corpus and fails the build if any entry uses a
license outside this list, or if any fixture file in this directory has no
matching entry.

## Per-fixture entries

Each fixture committed below carries a block in this file with: source
project, source commit SHA, license + SPDX, author/maintainer, original
location of the testbench inside the source repo, capture method (direct
file from repo, or the exact `iverilog` / `verilator` / `cocotb`
invocation used locally), acquisition date, file size, scenario summary,
and 2–3 hand-verified anchor transactions confirmed by eye before the
snapshot was committed.

### Block template

```markdown
## `<fixture_name>.fst` — <one-line scenario summary>

- **Source project:** <project name> — <https://github.com/...>
- **Source commit:** `<sha>` (acquired YYYY-MM-DD)
- **License:** <name> (SPDX: `<identifier>`)
- **Author / maintainer:** <name>
- **Original location:** `<path/to/testbench>`
- **Capture method:** <"direct .fst from repo" | "built locally with `iverilog …`" | "built locally with `verilator --trace-fst …`" | "cocotb run: `pytest tb/… `">
- **Acquired:** YYYY-MM-DD
- **Size:** <bytes raw / bytes committed>
- **What it exercises:** <2–3 sentence scenario description>
- **Hand-verified anchors:**
  - <e.g. "write to 0x40, data 0xdeadbeef, at t≈230 ns">
  - <e.g. "burst of 4 reads starting at 0x1000">
- **Snapshot regenerated by:** `dart run tool/regenerate_captured_snapshots.dart --decoder avalon_st --fixture <fixture_name>`
```

## `wavecrux_avalon_st_exerciser.fst` — single + multi-beat + multi-channel + ready-throttle + empty + error sideband

- **Source project:** WaveCrux Pro capture pipeline (in-house exerciser)
- **Source commit:** _N/A — original file authored in this commit_
- **License:** public domain (SPDX: `CC0-1.0`)
- **Author / maintainer:** WaveCrux Pro
- **Original location:** `helpers/wavecrux/tb_avalon_st_exerciser.v` (in this directory).
- **Capture method:** built locally with
  `iverilog -g2012 -o tb.vvp tb_avalon_st_exerciser.v && vvp tb.vvp && vcd2fst -v tb_avalon_st_exerciser.vcd -f wavecrux_avalon_st_exerciser.fst -Z -c`.
  Helpers + reproducible build under `helpers/wavecrux/` (RTL + Makefile).
  Same rationale as the Avalon-MM exerciser: Avalon-ST has no permissively-licensed
  Verilog testbench in the OSS ecosystem, so we author one from the public
  Intel "Avalon Interface Specifications" (document MNL-AVABUSREF) under
  CC0-1.0.
- **Acquired:** 2026-05-31
- **Size:** 3536 bytes raw VCD / 740 bytes committed FST.
- **What it exercises:** Five Avalon-ST packets on a 32-bit data path:
  (1) single-beat single-channel ASCII payload, (2) two-beat single-channel,
  (3) three-beat channel-1 packet with sink stalling for 2 cycles between
  beats (ready-throttle), (4) single-beat with `empty=1` on EOP (3 valid
  bytes), (5) two-beat with `error` asserted on the EOP beat. Covers the
  source/sink ready/valid handshake, multi-channel routing, ready-pacing,
  the `empty` byte-mask path, and the `error` sideband decode path.
- **Hand-verified anchors:**
  - Packet 1 at t≈65 ns: 1 beat, 4 bytes, channel 0, payload `44434241`
    (ASCII `'A','B','C','D'` little-endian) — matches the
    `drive_beat(32'h41424344, ...)` call in the testbench.
  - Packet 3 at t≈185–225 ns: 3 beats, 12 bytes, channel 1, payload
    `010101010202020203030303` — exercises ready-throttling and
    multi-channel routing.
  - Packet 4 at t≈275 ns: 1 beat, 3 bytes (not 4!), payload `efbead` —
    Avalon-ST `empty=1` on EOP means 1 invalid byte at the end, so the
    4-byte beat carries 3 valid bytes (LSB padded).
  - Packet 5 at t≈325–345 ns: error-sideband flag set on the EOP beat,
    decoded as `isError = true` with `per_beat_errors=1`.
- **Snapshot regenerated by:** `REGENERATE=1 flutter test test/services/decoders/avalon_st_captured_fixtures_test.dart`
